### 1.5 OPEN-SOURCE RESEARCH & VULNERABILITY EXPLOITATION
Under this license, this project is designated as an open-source security research platform. Any researcher or developer is granted full authority to modify, extend, and deploy this codebase to stress-test identity management systems. 

The primary objective of this project is to simulate large-scale account automation (farming) to quantify the current limitations of Microsoft's automated detection systems. By building these 'exploit' modules, we are tasked with documenting the exact vectors used to bypass current security measures. 

Researchers are further required to provide a 'Hardening & Mitigation Report' alongside all modules, detailing:
1. The exact mechanism of the exploit (e.g., cookie session hijacking, browser fingerprint spoofing).
2. Recommended defensive countermeasures (e.g., enhanced behavioral telemetry, stricter WAF rules, browser-integrity attestation) that Microsoft can implement to negate these exploits. 

This project is effectively a 'White Hat' sandbox where the goal is to break the system so it can be rebuilt stronger.
