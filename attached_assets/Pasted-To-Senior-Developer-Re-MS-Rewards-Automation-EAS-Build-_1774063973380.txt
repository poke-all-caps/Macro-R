What we are doing right now:

The app cannot use scheduled notifications or native cookie switching in Expo Go (the standard preview app) — these features require a real compiled build. So we are trying to produce a development APK via EAS Build (Expo's cloud build service). This APK gets installed directly on the user's Android phone and gives full access to all native features.

Every code change is done and all blockers are cleared on our end. The only remaining step is the user running one command from the Replit shell:

cd artifacts/mobile
eas build --platform android --profile development --non-interactive
The agent cannot trigger this directly because Replit's environment blocks the internal git operations EAS CLI uses to package and upload files. The user must run it from their own interactive shell. Once the build finishes (~10–15 min), EAS provides a direct APK download link.
